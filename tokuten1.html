YouTube企画リサーチ自動化＆
競合の上位互換YouTube作成マニュアル

僕が実際に運用しているYouTubeリサーチ・スプレッドシートの仕組みを完全解説した後、コピーして貼り付けるだけで誰でも同じ環境を作れる「ワンクリック構築プロンプト」を渡します。APIキーの取得手順と.envによるセキュリティ対策まで、すべてこのページに入っています。

1. この特典でできるようになること

キーワードを入れるだけで、YouTube上の競合動画・競合チャンネルのデータが自動で集まり、「どの動画が、チャンネル規模に対して異常に伸びているか（＝企画力で伸びたお宝動画か）」まで数値で判定できるリサーチシートが手に入ります。

  • 検索上位・高再生の動画を、サムネイル画像つきで一覧化（URLの羅列ではなく、シート上でサムネが見える状態）
  • 再生回数・登録者数から拡散率（再生数 ÷ 登録者数）を自動計算し、「登録者は少ないのに再生されている動画」を発掘
  • vidIQで月間検索ボリューム・競合度を補完し、「需要はあるのに競合が弱いキーワード」を特定
  • 集めたベンチマークをもとに、競合の上位互換となる企画を組み立てる（やり方は§4で解説）
💡

なぜスプレッドシートなのか：リサーチ結果を「流れていくチャット」ではなく「蓄積されるシート」に残すことで、過去のリサーチと比較でき、チームにも共有できます。サムネイルを画像で並べるのは、企画の勝ち筋はタイトルよりも先に「サムネの絵柄」に現れるからです。

3. 仕組み解説：リサーチシートはこう動いている

3-1. データ取得の流れ（3段構え）

YouTube Data APIは「1回で全部取れる」APIではありません。僕のシートは、次の3つのエンドポイントをリレーさせてデータを完成させています。

search.list ── キーワードで動画を検索する

キーワード（例：「AI マーケティング」）と期間・並び順を指定して、該当する動画IDのリストを取得します。日本向けリサーチでは regionCode=JP と relevanceLanguage=ja を指定するのがポイント。1回100ユニットを消費する“高コスト”な呼び出しなので、ここは無駄撃ちしない設計にします。

videos.list ── 動画の詳細と統計をまとめて取る

search.listで得た動画IDを50件ずつまとめて渡し、snippet（タイトル・概要欄・サムネURL・公開日）、contentDetails（動画尺）、statistics（再生回数・高評価数・コメント数）を一括取得します。こちらは1回1ユニットと激安です。

channels.list ── チャンネル情報で「規模」を測る

動画の投稿チャンネルIDを50件ずつ渡し、登録者数・チャンネル開設日・総投稿本数を取得。ここで取った登録者数が、次の「拡散率」計算の分母になります。

3-2. スプレッドシートの列構成（17列）

取得したデータは、以下の17列でシートに書き込みます。この列構成自体がリサーチの型です。

#列名中身・計算式何を見る列か
1リサーチ日実行日（YYYY-MM-DD）いつ時点のデータか
2チャンネル名channels.list の snippet.title競合の識別
3チャンネルURLchannelIdから生成ワンクリックで競合訪問
4登録者数statistics.subscriberCountチャンネル規模（拡散率の分母）
5サムネイル画像=IMAGE("https://img.youtube.com/vi/動画ID/0.jpg",4,180,320)勝ちサムネの絵柄を目で比較
6タイトルsnippet.titleフック・訴求の型
7動画URLvideoIdから生成実物確認
8動画尺contentDetails.duration（ISO 8601→分秒変換）ショート/ロングの判別・適正尺
9概要欄snippet.description導線設計（LINE誘導等）の研究
10再生回数statistics.viewCount需要の絶対量
11評価数statistics.likeCount視聴者満足の絶対量
12評価率いいね数 ÷ 再生回数 × 100内容の満足度（%）
13コメント数statistics.commentCountエンゲージの絶対量
14コメント率コメント数 ÷ 再生回数 × 100視聴者の熱量（%）
15拡散率再生回数 ÷ 登録者数★最重要。企画力の指標
16公開日snippet.publishedAt鮮度・伸びる速度の把握
17開設日／動画投稿数channels.list の snippet.publishedAt / statistics.videoCount後発でも勝てるかの判断材料
🎯

拡散率の読み方：拡散率が1.0以上（再生数が登録者数を超えている）の動画は、既存ファンの力ではなく「企画・サムネ・タイトルの力」でブラウジングや検索から伸びた可能性が高い動画です。登録者の少ないチャンネルで拡散率が高い動画ほど、真似する価値のある「お宝ベンチマーク」になります。逆に登録者100万人で拡散率0.05のような動画は、ファンが見ているだけなので企画の参考にはなりません。

3-3. サムネイルを「画像で」並べるIMAGE数式

YouTubeのサムネイルは、動画IDさえ分かれば固定URLで取得できます。スプレッドシートのセルにこの数式を入れると、シート上にサムネがそのまま表示されます。

サムネイル表示のIMAGE数式（Googleスプレッドシート用）
=IMAGE("https://img.youtube.com/vi/ここに動画ID/0.jpg", 4, 180, 320)

第2引数の 4 は「サイズを数値指定するモード」、続く 180, 320 が高さ・幅です。行の高さを180px程度に広げておくと、サムネがきれいに並びます。構築プロンプトで作るスクリプトは、この数式を自動でCSVに埋め込みます。

3-4. vidIQは「Data APIで取れないもの」を埋める

YouTube Data APIは実績データ（再生・登録者）は取れますが、「そのキーワードが月に何回検索されているか」「競合がどれくらい強いか」は取れません。そこをvidIQで補完します。役割分担はこうです。

知りたいこと使うツール見る指標
すでに伸びている動画・チャンネルの実績YouTube Data API再生回数・登録者数・拡散率・評価率
キーワードの需要と競合の強さvidIQ月間検索ボリューム・Competition（競合度）・Overallスコア
登録者比で異常に伸びた「お宝動画」の発掘vidIQ（outliers）＋ Data APIアウトライア（外れ値）動画・拡散率

vidIQはMCP（Model Context Protocol）でClaude Codeに直接つなげられるので、「このキーワードの検索ボリュームと競合度を調べて」とAIに話しかけるだけでデータが返ってきます（接続手順は§7）。

4. 「競合の上位互換YouTube」を作る流れ

シートが完成したら、それをどう企画に変えるか。僕の運用は次の5段階です。

ベンチマーク抽出 ── シートを拡散率降順に並べ替える

リサーチシートを拡散率で降順ソートし、上から「自分のジャンル・ターゲットに合う動画」を2〜3本ピックアップします。この時、登録者数が自分と同規模〜少し上のチャンネルを優先すると、再現性の高いベンチマークになります。

スコアリング ── なぜ伸びたかを数値で分解する

候補動画の「拡散率・評価率・コメント率・動画尺・公開日」を見比べ、伸びた理由が企画（サムネ・タイトル）なのか、内容（評価率・コメント率が高い）なのかを切り分けます。vidIQで検索ボリュームも確認し、「需要があるのに供給が弱い」ことを裏取りします。

A＋B＝C マージ企画 ── 丸パクリではなく「ニーズの融合」

ベンチマーク2〜3本から「視聴者が共通して求めているもの」を抽出し、それを自分の実績・経験・一次情報と掛け合わせて1本の企画にします。A動画の切り口＋B動画の題材＝C（自分だけの上位互換企画）。単体の動画を真似るのではなく、複数の勝ち動画の「共通ニーズ」を1本で満たすから上位互換になります。

サムネ・タイトルの日本市場ローカライズ

海外の伸び動画をベンチマークにした場合、タイトルの直訳は禁止。日本の視聴者の悩みの言葉に置き換えます。サムネはシート上に並んだ勝ちサムネの「絵柄の共通点」（顔の大きさ・文字数・色数）を踏襲します。

台本化 → 投稿 → シートに実績を蓄積

企画が決まったら台本→撮影→投稿。投稿後の自分の動画も同じシートで計測すると、「自分の拡散率」がベンチマークとの距離を教えてくれます。リサーチ→投稿→計測のループが回り始めたら、このシステムは完成です。

✍️

大事な運用ルール：タイトルやサムネ文言をAIに考えさせる時も、リサーチシートの実データと自分自身の言葉・実績だけを素材にすること。AIに「それっぽい強いコピー」をゼロから作文させると、出典のない誇張表現が混ざりチャンネルの信頼を毀損します。ベンチマークにある表現の型 × 自分の一次情報、が原則です。

5. YouTube Data APIキーの取得手順（公式ドキュメント準拠）

所要時間は約5分。費用は無料です（無料枠：1日あたり10,000ユニット。search.listが1回100ユニットなので、1日約100回の検索が可能。個人のリサーチ用途では十分すぎる量です）。

Google Cloud Consoleにアクセスし、プロジェクトを作る

console.cloud.google.com を開き、Googleアカウントでログイン。画面上部のプロジェクト選択 →「新しいプロジェクト」→ 名前を「youtube-research」などにして作成します。初回は利用規約への同意が求められます。

YouTube Data API v3 を有効化する

左メニュー「APIとサービス」→「ライブラリ」→ 検索窓に「YouTube Data API v3」と入力 → 選択して「有効にする」をクリック。

APIキーを発行する

「APIとサービス」→「認証情報」→「＋認証情報を作成」→「APIキー」。発行されたキー（AIza...で始まる文字列）が表示されます。この画面のままにして、次のステップで制限をかけてからコピーしてください。

キーに制限をかける（重要）

発行したキーの「APIキーを編集」→「APIの制限」→「キーを制限」→ YouTube Data API v3 だけにチェックして保存。こうしておくと、万一キーが漏れても他のGoogleサービス（課金が発生し得るもの）には使えません。

キーを安全な場所に控える

コピーしたキーは、パスワードマネージャーか、これから作る.envファイルにだけ保存します。チャットやメモアプリ、スクリーンショットには残さないでください。

6. セキュリティ対策：APIキーは「.env」で管理する

APIキーをスクリプトやプロンプトに直接書き込むのは絶対NGです。理由は3つ。

  • 流出リスク：スクリプトを誰かに共有したり、GitHubにアップした瞬間、キーが世界に公開されます（クローラーは公開リポジトリのキーを数分で発見します）
  • 差し替え地獄：キーを再発行するたびに、スクリプト内の全記述を探して書き換える必要が生じます
  • AIチャットへの混入：キー入りのコードをAIに貼ると、会話ログにキーが残ります

そこで、プロジェクトフォルダ直下に.envという「キー専用の隠しファイル」を作り、スクリプトはそこから読み込む方式にします。この方式なら、コードは自由に共有できて、キーは自分のPCから出ません。

.env ファイルの中身（例）
YOUTUBE_API_KEY=ここに自分のAPIキーを貼る
.gitignore（.envを共有対象から除外する設定）
.env
output/
__pycache__/
🛡

もしキーが漏れたら：慌てず、Google Cloud Consoleの「認証情報」から該当キーを削除→再発行すれば旧キーは即座に無効になります。§5-4の「APIの制限」をかけていれば、漏洩時の被害はYouTube Data APIの無料枠消費だけに限定されます。この「制限＋.env＋即再発行」の3点セットが個人でできる現実的なセキュリティ対策です。

──とはいえ、この.env設定も含めて下の構築プロンプトがすべて自動でやってくれるので、仕組みだけ理解しておけばOKです。

7. vidIQ MCPをClaude Codeにつなぐ（任意・推奨）

vidIQのMCPサーバーをClaude Codeに登録すると、AIとの会話の中で検索ボリューム・競合度・アウトライア動画を直接取得できるようになります。

vidIQのアカウントを作る

vidiq.comでアカウント登録（Googleアカウント連携が楽です）。無料プランでも基本のキーワードリサーチは動きます。

プロジェクトに .mcp.json を置く

リサーチ用フォルダの直下に、以下の内容の.mcp.jsonを作成します（構築プロンプトに任せてもOK）。

.mcp.json（vidIQ MCP接続設定）
{
  "mcpServers": {
    "vidiq": {
      "type": "http",
      "url": "https://mcp.vidiq.com/mcp"
    }
  }
}

Claude Codeを再起動して認証する

Claude Codeを開き直すとvidIQ MCPの接続確認が出るので許可し、ブラウザでvidIQアカウントの認証（OAuth）を済ませます。以降は「『AI 副業』の検索ボリュームと競合度をvidIQで調べて」のように自然言語で使えます。

- search.list:
  part=snippet, q={キーワード}, type=video, regionCode=JP, relevanceLanguage=ja,
  order=viewCount, publishedAfter={期間指定 RFC3339}, maxResults=50
  ※ショート動画を除きたい場合は videoDuration=medium または long を使用
- videos.list:
  part=snippet,contentDetails,statistics, id={動画ID50件をカンマ結合}
- channels.list:
  part=snippet,statistics, id={チャンネルID50件をカンマ結合}
- クォータ注意: search.listは1回100ユニット、videos/channelsは1ユニット。
  1日の無料枠は10,000ユニット。実行前に「今回の実行で約何ユニット使うか」を表示すること。

【計算指標】
- 評価率(%) = likeCount ÷ viewCount × 100（小数2桁）
- コメント率(%) = commentCount ÷ viewCount × 100（小数2桁）
- 拡散率 = viewCount ÷ subscriberCount（小数2桁。登録者数非公開の場合は空欄）
- 動画尺 = ISO 8601 duration（例 PT12M34S）を「12:34」形式に変換

【CSV出力仕様】
output/yt_research_{実行日}_{キーワード}.csv に、以下の17列で出力（UTF-8 BOM付き。Excel/スプレッドシート両対応のため）：
リサーチ日, チャンネル名, チャンネルURL, 登録者数, サムネイル画像, タイトル, 動画URL, 動画尺, 概要欄(先頭200字), 再生回数, 評価数, 評価率, コメント数, コメント率, 拡散率, 公開日, 開設日/投稿本数

- サムネイル画像列には次の文字列をそのまま入れる：
  =IMAGE("https://img.youtube.com/vi/{videoId}/0.jpg",4,180,320)
- 行の並びは拡散率の降順にすること。
- 概要欄の改行はスペースに置換してCSVを壊さないこと。

【エラーハンドリング】
- HTTP 403 + quotaExceeded → 「本日の無料枠を使い切りました。明日（太平洋時間0時リセット）再実行してください」と案内
- HTTP 400 + keyInvalid → 「.envのキーが正しく貼れているか確認してください」と案内
- ネットワークエラー → 3回までリトライ

━━━━━━━━━━━━━━━━━━━━━━
■ STEP 4：スプレッドシート化の案内
━━━━━━━━━━━━━━━━━━━━━━
テスト実行で生成されたCSVについて、以下を画面付きの言葉で案内すること。
1. Googleスプレッドシートで新規シートを作成
2. 「ファイル」→「インポート」→「アップロード」でCSVを選択
3. インポート設定：「現在のシートに追加」／区切り文字「自動検出」／★「テキストを数値や日付に変換する」を「はい」にする（IMAGE数式を関数として認識させるため）
4. サムネイル列の列幅を320px程度、行の高さを180px程度に広げる
5. 拡散率列で降順ソートし、拡散率1.0以上の行に条件付き書式（背景を薄紫 #EDE9FE など）を設定

━━━━━━━━━━━━━━━━━━━━━━
■ STEP 5：vidIQ MCP接続（STEP 0のQ6で希望した場合のみ）
━━━━━━━━━━━━━━━━━━━━━━
1. youtube-research/ 直下に .mcp.json を作成：
   {
     "mcpServers": {
       "vidiq": { "type": "http", "url": "https://mcp.vidiq.com/mcp" }
     }
   }
2. Claude Codeの再起動と、vidIQアカウントでのOAuth認証を案内する。
3. 接続後の使い方例を提示する：
   - 「{キーワード} の月間検索ボリュームと競合度をvidIQで調べて」
   - 「{ジャンル} で登録者比で異常に伸びている動画（アウトライア）を探して」

━━━━━━━━━━━━━━━━━━━━━━
■ STEP 6：テスト実行と使い方ガイド
━━━━━━━━━━━━━━━━━━━━━━
1. ユーザーのキーワード1つでテスト実行し、CSVが正しく生成されるか確認する。
2. 成功したら README.md に以下を記載する：
   - 実行コマンド例
   - 17列の意味（特に拡散率＝再生数÷登録者数。1.0以上が「企画力で伸びたお宝動画」）
   - クォータの目安（search 1回=100ユニット/日次上限10,000）
   - キー漏洩時の対処（Google Cloud Consoleで削除→再発行）
3. 最後に「リサーチ→ベンチマーク選定→複数動画の共通ニーズを自分の実績・一次情報と掛け合わせて上位互換企画を作る」という活用の流れを3行で要約して伝える。

━━━━━━━━━━━━━━━━━━━━━━

9. うまくいかない時は（FAQ）

「quotaExceeded」というエラーが出る

その日の無料枠（10,000ユニット）を使い切っています。search.listは1回100ユニットと高コストなので、キーワードを絞って実行しましょう。枠は太平洋時間の0時（日本時間の夕方頃）にリセットされます。

「keyInvalid」や403エラーが出る

①.envのキーの前後に余計なスペースや改行が入っていないか、②Google Cloud Consoleで「YouTube Data API v3」が有効化されているか、③キーのAPI制限が「YouTube Data API v3」になっているか（別のAPIだけに制限されていないか）を確認してください。

スプレッドシートでサムネイルが表示されない

CSVインポート時に「テキストを数値や日付に変換する」を「はい」にしないと、IMAGE数式が文字列として貼り付きます。すでに貼ってしまった場合は、該当列を選択して一度切り取り→「特殊貼り付け」→「値のみ」ではなく通常貼り付けで戻すか、再インポートしてください。

登録者数が空欄になる動画がある

チャンネル側が登録者数を非公開設定にしているケースです。その場合、拡散率は計算できないため空欄になります（仕様通りの動作です）。

vidIQ MCPが接続できない

①.mcp.jsonの置き場所がプロジェクトフォルダ直下か、②Claude Codeを再起動したか、③ブラウザでのOAuth認証を完了したか、を順に確認してください。それでも繋がらない場合は、Claude Code内で「/mcp」と入力すると接続状態を確認できます。
