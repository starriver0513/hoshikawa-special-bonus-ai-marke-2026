コンテンツマーケ効率化
API / MCP 一覧解説表＆導入手順書
（AI読み込ませOK）

僕がコンテンツマーケの効率化のために実際に使っているAPI・MCPを、個人情報を除いて網羅しました。それぞれ「なぜ使っているか」「どの作業・タスクで使っているか」を明記し、公式ドキュメントへのボタンを設置しています。ページ最後には、Claude CodeやCodexに貼ると導入を全部手引きしてくれる「導入ガイドプロンプト」を置いています。

1. この一覧表の読み方

  • API ＝ プログラム（スクリプト）からサービスを操作する接続口。スクリプトを書いて自動化する時に使う
  • MCP ＝ Model Context Protocol。AI（Claude Code等）に「手足」として直接つなぐ接続方式。設定すればAIとの会話だけでそのサービスを操作できる
  • 両対応のサービスは、まずMCPでつなぎ、複雑な自動化だけAPIスクリプト化するのが最短ルートです
  • 認証方式の目安：OAuth＝ブラウザでログインして許可するだけ／APIキー＝発行したキーを.envで管理（特典①§6と同じ方式）
🔐

導入前の約束：どのサービスでも、APIキー・トークンをチャットやスクリプトに直接書かず、.envファイルに保存して.gitignoreで除外してください。この手順は最後の「導入ガイドプロンプト」に組み込み済みです。

■ 進め方
━━━━━━━━━━━━━━━━━━━━━━
STEP 1｜ヒアリング（1問ずつ聞くこと）
  Q1. あなたのビジネスは何ですか？（例：講座販売／コンテンツ販売／受託／コーチング／物販）
  Q2. 今使っているSNS媒体・配信媒体は？（YouTube/X/Instagram/LINE/メルマガ等）
  Q3. 今いちばん時間を奪われている作業は？（リサーチ／投稿作成／CS対応／経理／レポート等）
  Q4. すでに契約しているツール・サブスクは？（ChatGPT/Claude/Notion/freee等）
  Q5. 使用するAIエディタは？（Claude Code／Cursor／Codex CLI）

STEP 2｜導入プラン提案
  - 回答をもとに、カタログから「最初に入れるべき3つ」「次の3つ」「今は不要」に分類して提案する
  - それぞれ「何の作業が」「どう楽になるか」を1行で添える
  - 月額・無料枠の目安も添える（不明なものは「公式で要確認」と正直に書く）

STEP 3｜1ツールずつ導入実行
  選ばれたツールごとに、以下のテンプレで案内すること：
  (1) アカウント登録・キー発行のURL（カタログのURLを明示。手順は公式ドキュメントの表記に従う）
  (2) 取得した認証情報の保存：プロジェクト直下の .env に KEY名=値 で保存させる
      ・キーの値をチャットに貼らせないこと。「.env に直接貼ってください」と案内する
      ・.gitignore に .env を必ず追加する
  (3) 接続テスト：キーの値を表示せずに接続確認だけを行うスクリプト or MCP接続確認
  (4) 最初の活用例を1つ実行してみせる（例：Trelloにテストカードを作る、Sheetsにテスト行を書く）

STEP 4｜完了レポート
  - 導入済みツールの一覧、.envに保存済みのKEY名（値は伏せる）、次に追加すべきツールをまとめる

━━━━━━━━━━━━━━━━━━━━━━
■ 禁止事項
━━━━━━━━━━━━━━━━━━━━━━
- 認証情報の値をチャット・ログ・ファイル名に出力しない
- カタログにないツールを推測で「使っている」と言わない（追加提案する時は「カタログ外の提案」と明示する）
- 存在しないAPIエンドポイント・料金を推測で断言しない（不明点は公式URLを案内する）
- ユーザーの確認なしに外部へのメッセージ送信・投稿・公開を実行しない

それでは、STEP 1 の Q1 から始めてください。
